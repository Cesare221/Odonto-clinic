Resumo executivo para liberação interna do sistema

Este documento consolida o status atual do projeto e apresenta, de forma objetiva, os pontos necessários para avaliação interna. O objetivo é facilitar a aprovação do sistema em uma leitura única, clara, organizada e profissional.

Interface desktop refinada Fluxo de agenda e recepção operacional Login, pacientes, atendimento e financeiro integrados Ambiente local de demonstração

Visão geral do que será aprovado

  • Sistema com estrutura separada em frontend e backend.
  • Login com identidade visual corporativa e ambiente interno em azul profissional.
  • Dashboard operacional com agenda, fila, confirmações e indicadores do dia.
  • Cadastro e consulta de pacientes, com fluxo de atendimento e prontuário.
  • Confirmação pública com opção de remarcação por pré-reserva, mantendo o controle sob responsabilidade da recepção.

Status atual

  • Lint: aprovado.
  • Testes automatizados: aprovados.
  • Build de produção: aprovado.
  • Bundle otimizado com code splitting.

Como rodar o projeto

  1. Abra a pasta raiz do projeto.
  2. Execute npm install.
  3. Configure o arquivo .env com a conexão do MongoDB.
  4. Execute npm run seed para criar as credenciais demo.
  5. Execute npm run dev.

Frontend: http://localhost:5173
Backend: http://localhost:5000

Comandos principais

  • npm run dev inicia frontend e backend em conjunto.
  • npm run frontend inicia apenas a interface.
  • npm run backend inicia apenas a API.
  • npm run backend:dev inicia a API com recarga automática.
  • npm run test executa os testes automatizados.
  • npm run build gera a versão de produção do frontend.

Fluxos recomendados para demonstração

  • Entrar como recepção e validar agenda, fila e confirmações.
  • Entrar como administrador e validar a visão completa, incluindo financeiro e usuários.
  • Abrir um paciente, conferir a ficha e o histórico clínico.
  • Validar a remarcação pública por meio do fluxo de confirmação.

Observações para aprovação

  • As credenciais acima são geradas pelo comando npm run seed.
  • Se o banco já possuir usuários com os mesmos perfis, o seed evita duplicidade.
  • Este material foi preparado para uso em ambiente local de homologação.
  • Recomenda-se validar a interface com o zoom do navegador em 100%.
